01
● O que é

Visão geral do sistema.

O Caserna Equaliza é um sistema local (FastAPI + SQLite) para equalizar um campeonato de kart rental dividido em três categorias — Base, Graduados e Elite.

O sistema processa os HTMLs exportados do cronômetro oficial (KGV Race Tracks), extrai voltas limpas, calcula estatísticas robustas e produz um rating Elo dinâmico cross-categoria. A partir desses dados, sugere subidas, descidas e equalizações justas de kart, considerando que pilotos correm em múltiplos traçados do mesmo kartódromo.

Objetivo

  • Eliminar vantagem de kart (sandbagging e equipamento descompensado).
  • Manter mobilidade justa entre categorias — ninguém fica preso por temporadas.
  • Dar transparência total: toda decisão deriva de uma fórmula auditável.
✓
Filosofia. O piloto compete contra o traçado e contra os pares — nunca contra o kart que sorteou. Variância de equipamento é absorvida pelo sistema, não pelo piloto.

03
● Ingestão

Upload & Parser.

Como os HTMLs do cronômetro viram dados estruturados.

Fluxo

  1. Usuário envia o HTML de prova exportado do KGV Race Tracks.
  2. O parser (BeautifulSoup) extrai circuito, categoria, piloto, kart e tabela de voltas.
  3. Cada linha vira um registro com tempo_segundos e horario.
  4. Pace laps (primeira + última) recebem is_pace_lap = 1 e são ignoradas nas estatísticas.
  5. Validação descarta arquivos malformados; insere em transação atômica.
!
Pace lap = volta de saída/retorno ao box, naturalmente mais lenta. Incluí-la distorce média, mediana e Elo.

Validações

  • Tempo entre 1s e 600s (exclui leituras defeituosas).
  • Mínimo de voltas por piloto (MIN_VOLTAS, default 5) para entrar no cálculo.
  • Categoria ∈ { Base, Graduados, Elite }.
  • Detecção de tempo por regex (não por índice fixo de coluna) — resistente a mudanças no HTML.

05
● Balanceamento

Equalização de karts.

Como o sistema detecta um kart "lento" ou "rápido" demais.

Após cada etapa, o tempo do piloto é normalizado pelo traçado (seção 7) e comparado a um baseline pessoal (mediana das últimas N etapas). Diferenças sistemáticas por kart indicam desequilíbrio.

# 1. Tempo base do piloto em ritmo limpo
base_p = mediana( tempos_limpos(piloto, últimas 3 etapas) )

# 2. Tempo esperado nesta etapa (corrigido por traçado)
esperado = base_p × fator_tracado

# 3. Gap observado no kart desta etapa
gap_kart = mediana( tempos_etapa ) − esperado

Regras de equalização

  • Gap positivo grande (+0,6s a +1,2s) → kart lento, candidato a manutenção.
  • Gap negativo grande (−0,6s ou mais) → kart rápido demais, sinalizado.
  • Gap dentro da banda (± 0,3s) → kart equalizado.
  • Gap só conta se o piloto tem baseline (N ≥ 3 etapas limpas).
i
O sistema usa mediana (não média) para resistir a outliers — toques, saídas de pista, pace laps.
06
● Rating dinâmico

Elo Ranking.

Rating contínuo, pairwise, cross-categoria e cross-traçado.

Fórmula base

E_a = 1 / (1 + 10 ^ ((R_b − R_a) / 400))
R_a' = R_a + K × (S_a − E_a)
VariávelSignificado
R_a, R_bRating atual dos pilotos A e B.
E_aProbabilidade esperada de A bater B.
S_aScore contínuo ∈ [0, 1] baseado no gap de tempo, não só vitória/derrota.
KFator de volatilidade. Alto na calibração (40), baixo em pilotos estáveis (16).

Pairwise por etapa

Em cada etapa, todo piloto é pareado com todos os outros que correram o mesmo traçado. O score S_a vem do gap normalizado:

gap = (t_b − t_a) / max_gap
S_a = clip( 0.5 + gap / 2, 0, 1 )
★
max_gap usa o percentil 85 (P85) dos gaps observados no campeonato. Um outlier gigantesco não infla o denominador e não achata todos os deltas.

Seed inicial

  • Rating inicial por categoria: Base = 1200, Graduados = 1400, Elite = 1600.
  • Primeiras 3 etapas: K alto (calibração rápida).
  • Após 3 etapas: K ajusta para 16 (suaviza oscilações).
07
● Traçados

Conversão entre traçados.

Como comparar tempos de layouts diferentes do mesmo kartódromo.

O mesmo kartódromo roda em diferentes traçados (curto, longo, invertido). Comparar tempos cruamente é errado. O sistema calcula um fator de conversão usando pilotos-ponte — aqueles que correram em ambos os traçados recentemente.

fator_A→B = mediana( t_B / t_A para cada piloto-ponte )

Regras

  • Mínimo de 3 pilotos-ponte para o fator ser confiável.
  • Os 5 melhores tempos limpos de cada piloto em cada traçado são usados.
  • Se IQR dos ratios for alto (>10%), o fator é marcado instável e emite alerta.
  • Ratios extremos (< 0.7 ou > 1.3) são descartados (pilotos com algum problema em uma das etapas).

10
● Decisão

Comitê de Decisões.

A última palavra é humana — o sistema só sugere.

Toda proposta (subida, descida, troca de kart) entra como decisão pendente. O organizador aprova, rejeita ou posterga, sempre com justificativa escrita. Cada ação fica registrada com autor e timestamp, formando uma ata auditável exportável em PDF.

✓
Nenhuma decisão é automática. Isso protege o campeonato contra erros de dados e casos-limite.

● Matemática

Estatística aplicada.

Conceitos usados nas fórmulas acima.

MétricaUso no sistema

● Calibração

Pilotos-Ponte.

A "cola" que conecta traçados e circuitos diferentes.

Um piloto-ponte é aquele que correu em dois traçados ou circuitos em um intervalo curto, com baseline semelhante nos dois. É a única forma robusta de calibrar o fator de conversão (seção 7) sem chute.

Critérios

  • Corridas nos dois traçados em ≤ 4 etapas de distância.
  • Baseline limpo: pelo menos 5 voltas válidas em cada.
  • CV < 3% (consistência) — piloto errático não serve de ponte.

● Zona de risco

Limpeza & segurança.

Operações destrutivas e como se proteger.

  • /api/limpar/etapas — apaga todas as etapas e voltas.
  • /api/limpar/equalizacoes — apaga histórico de equalizações.
  • /api/limpar/tudo — reset completo (mantém apenas configuração).
  • scripts/reparse.py — wipe + reconstruir a partir dos HTMLs em uploads/. Faz backup automático.
!
Sempre faça backup do arquivo SQLite antes. O sistema oferece um toast com 5 segundos de "Desfazer" após cada ação destrutiva — use.
